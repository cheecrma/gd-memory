# 건대의 기억 (가칭)

> **건대 졸업생이 만든, 건국대 캠퍼스에 기억과 발견을 남기는 AR 앱**

---

## 1. 한 줄 정의

캠퍼스의 특정 장소에 텍스트 기록을 남기면, 누구든 그 자리에서 카메라를 켰을 때 공중에 그 글이 떠 있는 위치 기반 AR 기억 보관소.

## 2. 문제와 기회

장소에 얽힌 기억(첫 고백, 입학식, 밤샘 과제)은 어디에도 기록되지 않고 휘발된다. 지도 앱은 POI 정보만 다루고, 커뮤니티(에브리타임)는 정보·익명 게시판일 뿐 "공간에 박제된 기억"이라는 영역은 비어 있다. 유사 컨셉(WallaMe 등)은 전국 단위 오픈 서비스로 출발해 콘텐츠 밀도 부족(빈 세계 문제)으로 실패했다. 본 서비스는 **좁은 면적 + 높은 일일 유동 + 반복 이벤트(입학·축제·졸업)** 조건을 갖춘 단일 캠퍼스로 한정하여 이 문제를 구조적으로 회피한다.

## 3. 핵심 컨셉

- **사람은 열고, 장소는 닫는다**: 사용자는 누구나(재학생·졸업생·주민·방문객), 기록 가능 구역은 건국대 캠퍼스 및 주변 일부로 GPS 펜스 제한. 상권 리뷰성 비방 리스크를 원천 차단.
- **콘텐츠 두 축**:
  - 💌 **기억형** — "여기서 처음 고백했던 곳" (쓰는 동기, 시간이 지날수록 가치 상승)
  - 📍 **발견형** — "호수 잘 보이는 명당", "벚꽃 피크는 4월 둘째 주" (읽는 동기, 즉시 효용)
- **창업자 내러티브**: "건대 졸업생이 모교에 만든 기억 보관소" — 커뮤니티 확산과 학교 제휴의 핵심 자산.

## 4. 사용 시나리오

| 사용자    | 시나리오                                   |
| ------ | -------------------------------------- |
| 재학생 커플 | 일감호 벤치에 기념 기록, 1년 뒤 같은 자리에서 다시 발견      |
| 신입생    | 산책 중 카메라를 켜고 선배들이 남긴 명당 팁·응원 메시지 발견    |
| 졸업생    | 졸업 후 방문해 과거 자신이 남긴 글 재회, 후배에게 메시지 남김   |
| 지역 주민  | "20년째 이 산책로로 출근" — 장소의 기억층을 두껍게 만드는 기록 |

## 5. 제품 설계 원칙

- **인증**: 카카오 로그인(저마찰 기본 관문) + 학교 메일/졸업증명 인증 시 "건국대 18학번" **선택적 배지**. 화면은 닉네임, 작성자 정보는 운영자 보관 — 추적 가능성 확보, 실명 노출 없음.
- **모더레이션 3중 구조**: ① 작성 화면 톤 설정(placeholder·예시로 콘텐츠 결 유도) ② 작성 시점 LLM 실시간 필터(욕설 + 실명 저격 패턴 감지) + 법적 책임 고지 ③ 신고 즉시 숨김.
- **데이터**: 텍스트 + GPS + 시각만 수집(이미지 배제) — 경량 운영, 개인정보 부담 최소화.
- **리텐션 장치**: "1년 전 오늘 이 자리의 글" 알림, 졸업식 한정 타임캡슐, 축제 한정 낙서존.

## 6. 기술 개요

- **측위/AR**: ARCore Geospatial API(VPS) — 야외 개방형 캠퍼스 특성상 미터 이하 정밀도 + 고도 확보 가능. 실내는 1차 범위 제외.
- **PoC 스택**: 웹AR(QR 진입, 앱 설치 불필요) + 카카오 OAuth + 텍스트 DB + LLM 필터 파이프라인.

## 7. 검증 계획 (PoC)

- **시점**: 가을 축제 또는 2월 졸업식 단일 이벤트.
- **준비**: 일감호 등 핵심 포인트 5~10곳에 운영자가 발견형 콘텐츠 사전 시딩(빈 세계 방지).
- **핵심 검증 지표**: 신기함(1회 작성)이 아니라 — **글을 남긴 사용자 중 "타인의 글을 보기 위해" 카메라를 다시 켠 비율 20~30% 이상.** 미달 시 중단, 달성 시 정식 운영 전환.

## 8. 리스크와 대응

| 리스크                   | 대응                                               |
| --------------------- | ------------------------------------------------ |
| 빈 세계(콜드스타트)           | 단일 캠퍼스 한정 + 이벤트 집중 출시 + 사전 시딩                    |
| 저격·비방 (Yik Yak 실패 패턴) | 추적 가능 계정 + 실시간 LLM 필터 + 법적 고지 + GPS 펜스(현장 방문 비용) |
| 학교 측 제지               | 동문 내러티브 기반 총학·홍보팀 제휴 우선 접촉                       |
| 읽기 동기 부족              | 발견형 콘텐츠 축 + 시간 기반 리텐션 장치                         |
| 대형 플랫폼 모방             | 단일 캠퍼스 밀도·동문 내러티브는 복제 곤란, 검증 후 타 캠퍼스 템플릿 확장      |

## 9. 확장 비전

건대에서 검증된 모델은 앱이 아니라 **템플릿**: "○○대의 기억" — 각 학교 졸업생 운영자와 함께 캠퍼스 단위로 복제 (하버드 → 전 대학으로 확장한 페이스북 경로). 장기적으로 "장소에 박제된 기억"이라는 신규 데이터 레이어 확보.
